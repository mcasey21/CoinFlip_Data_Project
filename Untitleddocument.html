Project report 

 

Meeting the brief 

For my project I made heads and tails which is a game of randomness as it has a chance to land on either face of the coin. To meet the basic requirement of having a random aspect in my game I import random python library.

 

The second basic requirement was to allow at least three inputs. I met this requirement as I asked which mode the player would like to play, single player, multiplayer or simulation mode. Within the single player mode, I allowed for another input by asking their guess. My multiplayer mode has even more inputs. It asked for two names and two guesses.  

 

 

 

My artefact meets the third basic requirement as it can be played in different modes. My code has three different modes, a single player mode, a multiplayer mode and a simulation mode. The single player mode works by asking the user to pick heads or tails. The coin flip function flips the coin then prints the result. If the guess and the coin flip match, then the user wins if not they lose. My multiplayer mode works by asking for the two players' names. These two players then have to input their guesses. The coin flip function I executed again. If one player guesses correctly, they win, if they both guess right, they draw otherwise they lose. The simulation mode loops one hundred times for each strategy. The strategies are only guessing heads and guessing randomly. The data from these modes is appended to the csv file.

 

 

 

I also met the advanced requirements for my project. The first advanced requirement asks me to extend my project so that it stores results and data in a csv file to be analysed. I did this by creating a function to create a csv file which opens the file and writes the headers in. The data from the simulation is then added to the csv file. I did this by opening the csv file I made earlier and appending data from certain variables. This resulted in a csv with rows of data to be analysed.  

 

 

The next advanced requirement asked me to analyse my data. I did this by importing the pandas library. I then added the data to a data frame. I took this data frame and turned it into two other sets of data. These other sets were the two strategies, tails and random. I then imported the matplotlib library. I used this to make two bar charts, one about the tail's strategy and one about the random strategy.  

 

The last requirement was to test my hypothesis. My hypothesis was, I want to find the most efficient way of winning and I want to see if I can break the 50/50 chance of the game. I met this requirement as I made graphs to prove that the game is 50/50. I also found out that the random strategy resulted in the most effective mode of play. I also asked what if questions such as what If I added a heads strategy.

Design 

I plan on making heads or tails for my project. My game will have two modes, multiplayer and single player. The different modes will be accessible via an interface that I will design. On this interface I will explain the rules of both modes and allow the user to pick their mode of choice.

I am going to use the python library matplotlib for graphing my results. I'm using this built-in module as it's easy to use and has unique features such as saving the graph to your computer. Random is another module that I am going to use. Random is a library that allows for random outputs. The brief states that my game must be random, so I must use this module. Pandas is a data analysis module made for python. It helps to clean and sort the data I will receive from my game. I will input my data into an excel spreadsheet to keep it organized.  

I am using an agile approach to design my project. I'm using this as iteration is a step that is revisited again and again. This repeated iteration helps me to fix issues before they become bigger problems. My project is also going to have a modular design. I'm using this design as each module is small, so problems and bugs are easier to fix.  

My code is going to have a while loop which will run until the user quits the game. I will also implement a function into my code so I can avoid rewriting the same code again and again in my program.

Computational thinking is the mental skill to apply concepts, methods and problem-solving techniques to solve problems in all areas, including our daily lives. Abstraction and decomposition are computational thinking methods that I am going to use during my creation on my project. Abstraction is the removal of any unnecessary information to make the problem easier to solve. Decomposition is when you break down the large task into smaller more manageable tasks. I'm using decomposition as the project is split into two big tasks, making the game and then analysing data from the game. I will make up certain tasks such as creating a function and getting the data output onto a spreadsheet. This will help the creation of my project as the smaller tasks are less daunting than the larger ones.

Evaluation 

At the start of my project, I stated that I wanted to apply different strategies to my game, and I wanted to see if I could break the 50-50 chances of heads and tails. I created two strategies for the game, a random strategy that changed randomly between heads and tails for 100 throws of the coin and a strategy that involved me only picking tails. I achieved what I set out to do as I examined strategies. I found that only choosing tails resulted in more draws than the random strategy and the random strategy generated more wins than the tails strategy, but what I found interesting was how the two strategies had around the same number of losses. Only the wins and draws seemed to deviate. In conclusion, the tails strategy resulted in more draws and less wins while the random strategy had less draws but more wins. From the results in my graphs, I believe that heads and tails is a 50-50 game.  

 

 

I enjoyed the project. I liked the overall challenge of it as it pushed me past my normal standard of code. The only part that I found difficult was the advanced requirement of adding my data to a csv file. I found this challenging as I had never done anything like this before. It was a learning curve for me. I enjoyed the making of the game itself as I ran into no problems making it. If I were to do this project again, I would change my rules so that it wasn’t your standard game of heads and tails. I would have liked for the coin to flip again if I got the answer right, this would result in me being able to add a point for every correct answer in a row. I would have then analyzed the high scores using different strategies. I would have also liked to add a heads strategy and a strategy where you guess heads for half and tails for the other half.  

Overall, I enjoyed doing this project and would love to maybe revisit it in the future and make the changes I talked about.
